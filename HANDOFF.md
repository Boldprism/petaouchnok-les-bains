## Failed Approaches (Don't Repeat These)

- **Single `tileset_master.png` with `tileoffset`**: Phaser doesn't support `tileoffset` in Tiled JSON. Each tileset must be a separate PNG file with its own `firstgid`. Switched to 4 individual PNGs (grass_forest, grass_path, path_plaza, grass_river) with firstgid 1, 17, 33, 49.

- **External tileset references (`"source"` field in JSON)**: Phaser can't resolve external `.tsx` tileset files. All tileset definitions must be embedded inline in the Tiled JSON.

- **Loading nonexistent `source.png` and `halo_gold.png`**: Crashed Phaser preload silently (no error, just green rectangle). Source thermale is now drawn entirely via Phaser Graphics API — no sprite needed.

- **Relative asset paths (`assets/map/...`)**: Must be absolute (`/assets/map/...`) for Next.js public folder serving.

- **Multiple Wang tile index iterations**: The corner-based lookup uses (NW, NE, SW, SE) corners where each corner checks `(cx,cy)`, `(cx+1,cy)`, `(cx,cy+1)`, `(cx+1,cy+1)` — NOT the 8-neighbor approach. The correct WANG lookup table is:
  ```python
  WANG = {
    (0,0,0,0):6, (0,0,0,1):7, (0,0,1,0):10, (0,0,1,1):9,
    (0,1,0,0):2, (0,1,0,1):11, (0,1,1,0):4, (0,1,1,1):15,
    (1,0,0,0):5, (1,0,0,1):14, (1,0,1,0):1, (1,0,1,1):8,
    (1,1,0,0):3, (1,1,0,1):0, (1,1,1,0):13, (1,1,1,1):12
  }
  ```

- **`destroy(true)` for Phaser cleanup**: Caused issues on re-mount. Must use `destroy(true, false)` (second param prevents removing the canvas from DOM prematurely).

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| 4 separate tileset PNGs, not tileset_master | Phaser needs individual images per tileset, no tileoffset support |
| Source thermale via Graphics, not sprite | No sprite asset exists; gold halo + pulse looks good procedurally |
| `key="phaser-map-instance"` on PhaserGame | Forces fresh Phaser instance when navigating back to /map |
| `min-h-0` on main, `overflow-y-auto` on pages | Proper flex overflow — main is flex container, pages scroll individually |
| Map uses `flex-1` not fixed `h-[calc()]` | Adapts to any device without hardcoded BottomNav height |

## Code Context

**Tileset loading in MapScene.js** (preload + create):
```js
// PRELOAD — 4 individual tilesets
this.load.image('ts_grass_forest', '/assets/map/grass_forest.png');
this.load.image('ts_grass_path',   '/assets/map/grass_path.png');
this.load.image('ts_path_plaza',   '/assets/map/path_plaza.png');
this.load.image('ts_grass_river',  '/assets/map/grass_river.png');

// CREATE — map name → Phaser key, all layers get ALL tilesets
const tsForet   = this.map.addTilesetImage('grass_forest', 'ts_grass_forest', 32, 32, 0, 0);
const allTilesets = [tsForet, tsChemins, tsPlace, tsRiviere].filter(Boolean);
this.layerForet = this.map.createLayer('Forêt', allTilesets, 0, 0);
```
